Slide 1 · Title

Private x402 payments for the metered agent economy.

WhisperAPI is the checkout layer that lets agents pay for APIs, quotes, and merchant services on Solana without exposing provider choice, spend, or usage cadence on public rails.

One-line analogy

Stripe Billing for agent-to-API payments, but private by default with MagicBlock PER.

Commercial handshake x402-compatible 402 -> private settle -> receipt retry
Private execution MagicBlock deposit + private transfer + optional withdraw
Target wedge Paid APIs, quote endpoints, merchant connectors
Presenter notes: Start with the leak. Today every public machine payment reveals which provider an agent uses, how much it pays, and how often it pays. WhisperAPI removes that exhaust without changing the basic x402 developer flow.
Slide 2 · Problem

Public payment rails turn agent spend into competitive intelligence.

What leaks today

  • Per-call amount is visible or easily inferable from the payment route.
  • Provider selection becomes readable once a wallet-to-service mapping is known.
  • Payment cadence exposes traffic, model usage, and supply strategy.

Why it matters

  • API providers, brokers, and merchant connectors become involuntary public dashboards.
  • Agents cannot negotiate or route privately if every payment leaves public exhaust.
  • Serious businesses will not move sensitive machine spend onto fully transparent rails.
Presenter notes: This is not “privacy for privacy’s sake.” It is an operational requirement for real machine commerce. If agents are going to buy search, compute, data, and travel on their own, the payment path cannot leak the playbook.

Presenter notes: The category shift is that autonomous software is now a buyer. The infrastructure shift is that private payment execution is now accessible without building bespoke privacy cryptography from scratch.
Slide 4 · Product

WhisperAPI wraps a standard paid request in a private settlement path.

01

Agent hits a paid endpoint

The endpoint returns 402 Payment Required and the payment specification.

02

WhisperAPI opens a private session

The app builds and signs a MagicBlock deposit transaction for the buyer wallet.

03

Private transfer settles inside PER

The provider destination is paid privately rather than through a readable public per-call transfer.

04

Receipt retries the original request

The provider returns the response, and the observer panel shows what did and did not leak.

Presenter notes: Judges should understand the flow in one breath. We are not replacing x402. We are making the payment execution part private while preserving the commercial handshake and the provider confirmation flow.
