Why now

AI agents need private machine payments, not public exhaust.

The product is simple to explain: x402 is the commercial handshake, MagicBlock PER is the private execution layer, and WhisperAPI turns them into a merchant-safe checkout path for agents.

Agent commerce

Paid APIs are the first recurring spend category for autonomous software.

Inference, search, maps, travel quotes, and merchant APIs all become pay-per-use infrastructure.

MagicBlock fit

PER is purpose-built for private state and compliant execution on Solana.

WhisperAPI uses the Private Payments API directly instead of wrapping privacy around the edges.

Hackathon shape

Architecture

A clean four-step payment path.

One flow for the buyer agent, one settlement destination for the provider, and one observer panel for judges.

01

Agent hits a paid endpoint

The API returns `402 Payment Required` with the payment spec and retry contract.

02

WhisperAPI opens a private payment session

The app builds a MagicBlock deposit transaction and moves the buyer balance into PER.

03

Private transfer settles inside PER

The provider destination receives the private payment without exposing the per-call route on public rails.

04

Receipt retries the original request

The API returns the product response, while observers only see what the selected settlement mode reveals.

Devnet setup

Paste your key, verify balances, then run the real flow.

The project now auto-loads `.env.devnet`, so you can configure devnet without shell gymnastics.

1. Fill the env file

Paste your buyer private key into `.env.devnet`, then set the provider destination and real devnet mint.

File path: D:\frontier hackhton\whisperapi\.env.devnet

2. Run the check script

Use `npm run check:devnet` to verify MagicBlock health, wallet readiness, and token balances before the UI demo.

3. Demo it live

Once `ready: true` appears in Integration Status, run the private flow and use the returned signatures in your demo.
