// The leak problem

Public payments broadcast
your playbook.

Every on-chain machine payment today leaves a readable trail. That's competitive intelligence handed to anyone watching.

Without WhisperAPI

Public payment trail leaks everything

  • Which API provider you use — inferable from the payment route
  • Exact spend per call — visible or reconstructable from amounts
  • Call frequency & cadence — exposes traffic and model usage
  • Agent strategy — merchant selection reveals intent
  • Budget ceiling — max payments expose negotiation position
With WhisperAPI

Settlement happens inside MagicBlock PER

  • Provider identity concealed — no wallet-to-service mapping
  • Amount stays private — moves inside ephemeral rollup
  • Frequency hidden — batched settlement only
  • Agent intent protected — checkout path is private by default
  • x402 flow preserved — providers see paid, not how much

// Payment flow

Four steps.
Zero leakage.

Standard x402 handshake on the outside. Private MagicBlock execution on the inside.

01
Agent hits paid endpoint

API returns 402 Payment Required with payment spec — amount, mint, cluster, settlement mode.

HTTP 402 → payment spec
02
WhisperAPI opens private session

Buyer funds deposit into MagicBlock PER. Session state and concealment flags are set. Mint initialized if needed.

POST /v1/spl/deposit
03
Private transfer settles

Provider destination receives private payment inside PER. No readable per-call trace on public Solana. Optional withdrawal to base chain.

POST /v1/spl/transfer
04
Receipt unlocks the API

Single-use receipt token retries the original request. Provider returns paid response. Observer sees only what settlement mode allows.
